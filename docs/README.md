# Documentation

## Purpose
- Central reference for rules, requirements, architecture, tech stack, data schemas, audits, and plans that guide the Crossfire Sector digital adaptation.

## Contents
- `rules.md` – Tabletop rules source.
- `project.md` – High-level project summary and goals.
- `requirements/` – Requirements set traced to game rules (GR/DA/CP/AQ/MP).
- `architecture/` – Architecture overview and tech stack alignment.
- `data-definition/` – Data dictionary and schema governance.
- `audits/` – Coverage and consistency audits.
- `plans/` – Roadmaps and implementation planning.

## Conventions
- Use ordered headings and hyphenated lists for consistency.
- Keep terminology aligned with rules (Move, Range, AQ, Defense) and requirements IDs when referencing traceability.
